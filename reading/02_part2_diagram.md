# Parte 2 — Reading to Apply a Diagram

## Contexto Típico

- Você recebe um **diagrama** + um email ou mensagem curta que se refere a ele
- Sua tarefa é cruzar as informações dos dois para responder as questões
- **8 questões** | ~9 minutos

**Tipos de diagrama que aparecem:**
- Tabela de horários / cronograma de eventos
- Tabela comparativa (preços, recursos, duração entre opções)
- Mapa ou planta baixa (localização, direções, posições)
- Fluxograma de processo (etapas sequenciais)

**Formato:**
- Q1–5: fill-in-the-blank completando um email com dados do diagrama
- Q6–8: múltipla escolha cruzando texto + diagrama

---

## O que Mais Cai

- Encontrar o valor correto em uma célula da tabela (linha certa + coluna certa)
- Aplicar uma condição do texto ao diagrama (ex.: "o mais barato que inclua X")
- Identificar localização relativa em mapas ("ao norte de / à esquerda de / próximo a")
- Sequência de etapas em fluxogramas
- Exceções marcadas com asterisco ou nota de rodapé no diagrama

---

## Regra de Ouro desta Parte

**Estude o diagrama ANTES de ler o email e ANTES de ver as questões.**

O diagrama é compacto e rápido de entender. Construa uma imagem mental dele em 30–45 segundos:
- Leia o título ou cabeçalho
- Leia todos os rótulos de colunas e linhas
- Leia a legenda, asteriscos e notas de rodapé
- Identifique as unidades (horas, $, km, m²)

Só depois abra o email.

---

## Palavras-Gatilho no Email para Prestar Atenção

Essas palavras introduzem condições que você precisa verificar no diagrama:

| Palavra / expressão | O que sinaliza |
|--------------------|----------------|
| "only if / unless / except" | condição restritiva — elimina opções do diagrama |
| "at least / no more than / up to" | limite numérico — filtra o diagrama |
| "before / after / between" | restrição de tempo ou sequência |
| "the cheapest / the fastest / the closest" | superlativo — buscar valor extremo na tabela |
| "if... then" | condição lógica — verifique se o diagrama satisfaz |
| "based on the schedule / according to the chart" | ponto de cruzamento texto → diagrama |

---

## O que Anotar no Papel

Para tabelas comparativas, anote as condições do email antes de abrir o diagrama:

```
CONDIÇÕES DO EMAIL:
  $ máx: ___________
  tempo: ___________
  deve incluir: _____
  não pode incluir: _
  outro: ___________
```

Depois aplique essas condições linha por linha na tabela — elimine quem não satisfaz.

Para mapas, defina um **ponto de referência fixo** (ex.: entrada principal, rua central) e localize tudo em relação a ele.

Para fluxogramas, numere mentalmente cada etapa: 1→ 2→ 3→

---

## Estratégia de Resposta (passo a passo)

1. **30–45s:** estude o diagrama — título, rótulos, legenda, asteriscos
2. **Leia o email** e identifique as condições/restrições que ele impõe
3. **Para Q1–5 (fill-in-the-blank):** leia a frase ao redor de cada lacuna, identifique que informação está faltando, localize no diagrama
4. **Para tabelas:** trace a linha E a coluna até a interseção — nunca leia só a linha
5. **Para Q6–8 (múltipla escolha):** elimine qualquer opção que viole uma condição do email, mesmo que o diagrama permita

---

## Armadilhas e Erros Comuns

| Armadilha | Detalhe |
|-----------|---------|
| **Lógica do mundo real** | Você "sabe" que trem é mais rápido que ônibus — mas se o diagrama mostra o contrário, o diagrama vence |
| **Ignorar asteriscos e rodapés** | Quase sempre contêm a resposta de pelo menos uma questão |
| **Rótulos parecidos** | "Economy Plus" vs "Economy" — confundir os dois é o erro mais comum em tabelas |
| **Linguagem condicional** | "Only if / unless / except" muda completamente se uma opção é válida — leia devagar nessas palavras |
| **Responder de memória** | Sempre volte ao diagrama para confirmar — nunca responda pelo que "acha que leu" |
| **Só verificar linha ou só coluna** | Para tabelas você precisa dos dois: linha + coluna → interseção |

---

## Símbolos para usar enquanto lê o email

| Símbolo | Quando usar |
|---------|-------------|
| ⚠ | condição restritiva (only if / unless / except) |
| $ | limite de custo |
| ✗ | elimina essa opção do diagrama |
| ✓ | opção que satisfaz todas as condições |
| → | "leva a" / próxima etapa do processo |
| [ ] | nota de rodapé ou asterisco importante no diagrama |
