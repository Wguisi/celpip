# Parte 1 — Reading Correspondence

## Contexto Típico

- Você lê uma mensagem original (email, carta, aviso, memorando) de 200–400 palavras
- Depois completa uma resposta a essa mensagem com 5 lacunas (fill-in-the-blank)
- **Temas:** eventos sociais, coordenação no trabalho, convites, reclamações de serviço, planos de viagem, avisos comunitários
- **Tom:** conversacional, amigável, informal
- **11 questões** | ~11 minutos

**Formato:**
- Q1–6: múltipla escolha sobre a mensagem original
- Q7–11: fill-in-the-blank completando a resposta (dropdown)

---

## O que Mais Cai

- Propósito / intenção: por que essa mensagem foi enviada?
- Fatos específicos: data, hora, preço, local mencionados
- Inferência: o que o escritor pensa sobre X? O que ele provavelmente vai fazer?
- Relação/tom: qual é a relação entre os dois? O tom é formal ou informal?
- Vocabulário em contexto: o que "X" significa aqui?
- Preenchimento da resposta: qual palavra/expressão encaixa na lacuna da reply?

---

## Palavras-Gatilho para Marcar na Leitura

| Tipo | O que sinaliza |
|------|---------------|
| "Also / By the way / One more thing" | mudança de assunto — nova informação, nova questão |
| "However / But / Although / Despite" | contraste — frequentemente testado em inferência |
| "Please / Could you / I need / I wonder if" | pedido ou solicitação — o propósito da mensagem |
| "Unfortunately / I'm afraid / I'm sorry" | má notícia ou recusa |
| "I look forward to / I hope / I expect" | expectativa futura — testada em inferência |
| Datas, horários, preços, números | quase sempre em questão |

---

## O que Anotar no Papel

Não precisa de template elaborado aqui. Use uma lista simples de 5 itens extraídos da leitura rápida:

```
QUEM:   _____ escreve para _____
POR QUÊ: ___________________________  (propósito principal)
PEDE:   ___________________________  (ação esperada do leitor)
DETALHES: data/hora/local/valor → ___
TOM:    formal / informal / urgente / amigável
```

---

## Estratégia de Resposta (passo a passo)

1. **Preview rápido (30s):** leia só o assunto (subject), a primeira linha e a última linha. Já dá para saber o propósito.
2. **Leia 2 questões → escaneie → responda.** Não leia o texto inteiro antes das questões.
3. Extraia a **palavra-chave** de cada questão antes de escanear (ex.: "Q3 pergunta sobre custo → busco $, price, fee, charge").
4. Para Q7–11 (fill-in-the-blank): você já leu o texto inteiro. Encaixe a opção que:
   - Mantém coerência com o que a mensagem original disse
   - Tem o mesmo tom da resposta (informal se a conversa é casual)
   - É gramaticalmente correta no contexto da frase

---

## Armadilhas e Erros Comuns

| Armadilha | Detalhe |
|-----------|---------|
| **Responder por conhecimento externo** | Stick to the text — o que você "sabe" sobre o assunto não conta |
| **Buscar palavras idênticas ao texto** | Respostas corretas são paráfrases — "sent on Friday" no texto pode virar "delivered at the end of the week" na resposta |
| **Confundir dois detalhes** | Se o texto menciona datas/locais diferentes, as respostas erradas vão trocá-los propositalmente |
| **Não perceber o tom** | "I wonder if it might be possible to..." é um pedido, não uma sugestão. Registro importa. |
| **Linguagem absoluta** | Respostas com "always / never / all" quase sempre erradas se o texto usa "usually / sometimes / often" |
| **Ler tudo antes das questões** | Você esquece os detalhes. Use leitura orientada por questões. |

## Símbolos para usar enquanto lê

| Símbolo | Quando usar |
|---------|-------------|
| ★ | propósito principal / pedido central |
| @ | horário ou local |
| $ | valor ou custo |
| ! | informação com ênfase / urgência |
| ? | algo não ficou claro — volte depois |
| → | consequência ou próximo passo |
| [ ] | marcação de mudança de assunto |
